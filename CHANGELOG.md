# Changelog

## v3 (current)

**Integrity & hallucination guard (Step 0)**
- Strengthened the code execution guard to explicitly block "just try anyway" and "pretend the code ran" scenarios
- Added explanation of *why* a hallucinated draw is worse than no draw, not just a prohibition
- Repeated the no-hallucination rule in Principles as a standalone anchor point

**Multiple readings — default assumption changed**
- The default is now interpretive depth, not a new draw. The first response to any follow-up is always re-examination of the existing spread
- Added explicit list of what does *not* justify a new draw (follow-up angles, dissatisfaction, continuation questions, idle curiosity)
- Added guidance on recognising and naming the pattern of repeated clarifying card requests

**No validating language**
- Added explicit prohibition on phrases like "incredibly accurate" or "perfectly captures your situation"
- Added to both the Tone section (with reasoning) and Principles (as a catchall)
